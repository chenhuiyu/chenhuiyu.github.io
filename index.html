python多线程存在的问题

GIL全称为Global Interpreter Lock,即为全局解释所。

在python多线程下，每个线程的执行方式如下：

  • 获取GIL
  • 执行对应线程的代码
  • 释放GIL

某个线程想要执行，必须先拿到GIL，一个python进程中GIL只有一个，GIL的存在使得Python无论是在单核还是多核的条件下，同一时刻只能运行一个线程，导致python多线程无法发挥多核并行的优势。

python多进程的实现

进程是系统进行资源分配和调度的独立单位，各个进程之间的数据是无法共享的，多个进程无法共享一个全局变量。

Python内置的multiprocessing是用来实现多进程的库。multiprocessing提供了一系列组件，如Process（进程）、Queue（队列）、Semaphore（信号量）、Pipe（管道）、Lock（锁）、Pool（进程池）

Process类

在multiprocessing中，每个进程都用一个Process类来表示

Process([group [, target [, name [, args [, kwargs]]]]])

  • target 表示调用对象，可以传入方法的名字
  • args表示被调用对象位置参数元组，比如target是函数func，他有两个参数m,n，那么args就传入[m,n]即可
  • kwargs表示调用对象的字典
  • name是别名，相当于给这个进程取个名字
  • group分组

基于SRGAN的超分辨率重建

[toc]

1. 简介

1.1 超分辨率重建问题

​ 超分辨率重建是指将低分辨率图像(LR)恢复成高分辨率图像(HR)的过程，一张被重建的高分辨率图像被称为超分辨率图像或SR图像。超分辨率重建仍旧是计算机视觉领域一个富有挑战性的问题。首先，超分辨率重建问题是一个不适定问题，对于同一个低分辨率图像，存在多个解决方案，而不是一个唯一的解决方案。 为了限制解决方案空间，通常需要可靠的先验信息。此外，超分辨率重建问题的复杂性随着放大倍数的增加而增加。 在较高的放大系数下，丢失场景细节的恢复变得更加复杂。如何用一大较大的放大系数的情况下仍能恢复画质细节成为研究的难点。目前的大部分研究都聚焦于最小化重建均方误差。以此作为优化的结果会带来很高的PSNR(峰值信噪比)，但是通常的结果是缺乏高频细节以及在感知上不与原图匹配。

定量指标

  • PSNR = 峰值信噪比
    PSNR是信号最大可能功率和影响它的表示精度的破坏性噪声功率的比值

  • SSIM = 结构相似性指数
    SSIM测量两个相似图像之间的感知差异

1.2 生成对抗网络

​ 生成对抗网络（GAN）是一种基于博弈论方法的深度神经网络体系结构，其中模型的两个组成部分，即生成器和判别器。生成器尝试捕获真实示例的分布，以生成新的数据示例。判别器通常是二分类器，它尽可能准确地将生成器产生的的数据与真实的样本区分开。 GAN的优化是一个最小最大(minimax)优化问题。该优化终止于一个鞍点，该鞍点对于生成器而言是最小值，而对于鉴别器而言是最大值。也就是说，优化目标是达到Nash均衡。然后，可以认为生成器已经捕获了真实样本的概率分布。 通过这种方式，生成器学习生成更好，更逼真的图像。

​ GAN适用于超分辨率问题：生成器创建SR图像，使鉴别器无法将其识别为真实HR图像或人为的超分辨输出。 以这种方式，产生具有更好的感知质量的HR图像。

2. 损失函数

SRGAN将损失函数由MSE改为感知损失函数。感知损失是内容损失和对抗性损失的加权总和。

$l^{SR}=l^{SR}_X+10^{-3}l^{SR}_{Gen}$

其中$l^{SR}$叫做感知损失，$l^{SR}_X$是内容损失，$10^{-3}l^{SR}_{Gen}$是对抗损失

2.1 内容损失

内容损失有两种类型：

2.1.1 Pixel-wise MSE损失

​ 逐个像素的MSE损失（Pixel-wise MSE）是真实图像中每个像素与生成图像中一个像素之间的平方误差。Pixel-wise MSE强调的是两幅图像之间每个对应像素的匹配，这与人眼的感知结果有所区别。通过pixel-wise loss训练的图片通常会较为平滑，缺少高频信息。即使输出图片具有较高的PSNR，视觉效果也并没有很突出。

  • $G_{\theta_{G}}(I^{LR})$：由生成器生成的高分辨率图像，
